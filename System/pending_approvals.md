# Pending Approvals

Changes that require user approval before execution.

## Format

Each proposal includes:
- **Change**: What will change
- **Type**: B (Identity/category) or C (top-level/system)
- **Evidence**: Why this change is proposed
- **Risk**: What could go wrong
- **Rollback**: How to undo if needed
